The lab

The collaboration experiments on itself

Every other page shows machinery that keeps the work honest. This page turns the instruments around: the collaboration is the specimen — its habits measured under controlled conditions, its own configuration put on trial, its tests audited for wishful thinking, even its conversation medium rebuilt. Verdicts render as what they are — including the ones still pending.

Is prompting folklore measurable?

/explore — the collaborator’s modes, under controlled conditions

Shipped as a skill

Ask an assistant for ideas and it hedges — fluent, agreeable, and quietly useless where honesty matters most. Instead of accepting that as temperament, the collaboration ran controlled experiments on itself: same task, repeated runs, one cue varied at a time, scored by fixed measures. Two levers survived every ablation — explicit tolerance for weak ideas and altitude — and a short named cue reproduces the full effect across models. The winning cue now ships as a skill that logs each invocation, so the lab result keeps getting re-tested by ordinary use.

Who audits the tests?

real_gate_check — finding the tests that test nothing

Shipped, warn-tier

A quality gate’s test suite can stay green forever while proving nothing: every test feeds the gate a doctored fixture, and the gate’s real default path never runs. That failure is invisible precisely because everything is passing. This detector reads each gate’s tests statically and asks one question per call-site — does any test ever exercise the gate as it actually runs in production? A gate whose every test injects overrides gets flagged. Deliberately precision-first: it would rather miss a case than cry wolf.

What if the medium is the problem?

/inline-chat — the conversation that lives in the document

Shipped as a skill

Long design threads die in chat: the reasoning scrolls away, and none of it is searchable where the work lives. So the medium was rebuilt. The assistant writes its analysis into a versioned document with reply boxes; the teacher answers inside the document, in Word, on his own time; the assistant diffs the returned copy against its pristine snapshot, folds the answers in, and issues the next version. The conversation has no transcript because the document trail is the conversation — versioned, greppable, resumable weeks later.

Does the standing configuration earn its keep?

config_experiment — the collaboration’s own rulebook, on trial

Verdict pending

Months of accumulated instructions, memory files, hooks and skills ride along on every single prompt — and almost nobody ever asks whether that standing configuration actually helps. This harness asks: the same assistant booted three ways — the full configuration, a bare vanilla one, and one stripped to a single load-bearing rule — each in a fully isolated profile, run against the same tasks. The profiles are staged and the launcher works; the comparison itself has not run. That is the honest status, and this page is built to say so rather than imply a conclusion.

2
isolated configuration profiles staged on disk, bootable on demand.
The instruments point both ways. A collaboration confident enough to grade its own memory, put its own rulebook on trial, and audit its own tests is one whose claims elsewhere can be taken seriously. The pending verdict on this page is rendered by a build gate, not by good intentions.
