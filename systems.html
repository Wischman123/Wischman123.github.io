The library learns

Drag the handle. On the left is a diagram the engine drew before a library fix; on the right, the same problem after. Nothing about this one page was hand-corrected — the fix landed in the shared diagram library and every affected problem re-derived. Slide back and forth to compare.

Why regenerate instead of patch

A one-file correction fixes one artifact and leaves every sibling wrong. The platform's rule is the opposite: a fix is a change to a shared library function, after which every artifact that depended on it re-derives and old versions retire. The before/after here is a single pair, but the same regeneration ran across the whole affected set — the library learned once, and everything downstream inherited it.

The gate says no

The system drafts a problem; it will not ship one that fails a check. Here is a real rejection, replayed: a generated problem is turned away by a completeness gate, the fix lands in the library, the problem is regenerated, and only then does it pass. The transcript is readable with JavaScript off — the animation is just typing on top of text that is already there.

Electric Field Lab

Drag either charge. The electric field lines re-trace as you move it — this is not a picture, it is the same Electric field line integrator the platform uses to draw diagrams, running in your browser. Select a charge and use the ← → ↑ ↓ keys to nudge it; Tab switches which charge is focused. Drag the two together and the lines stay well-behaved — a minimum-separation clamp keeps the field finite.

Idle until you touch it — nothing animates on its own.

Drag a charge, tap it to set its strength (−3 to +3), or click empty space to add another. Select a charge and press Delete to remove an added one; the + / − keys change its charge.

How the live math is pinned to the library

The tracer in assets/field_tracer.js is a port of the platform's own reference implementation, physics/lib/charges.py’s compute_field_line — the same fixed-step Euler integration along the unit field direction. A build-time check (tools/test_field_tracer.py) drives this browser tracer from the parameters recorded in a charges.py-generated fixture and asserts the polylines match within tolerance, so the interactive exhibit cannot silently drift from the library it demonstrates. When it last ran, the two agreed to the last floating-point bit.

A simulator, in the page

Three live scenes from the platform's physics simulator, embedded as lab instruments. Each rests on a static poster until you press Run scene — nothing moves offscreen or before you ask. Once running, click the scene to pause or resume it, or press ↺ to replay from the start. All three are genuinely different simulations: a two-body Coulomb orbit on the left, an underdamped RLC circuit on the right, and below them a spacecraft firing a single engine burn.

The third scene is the one my students actually get on paper. The spacecraft circles Ceres, fires a single forward burn, and the circle opens into an ellipse; it coasts out to the far point, where a piece of debris runs into it and sticks. That is a homework problem from the printed set — and the simulation is not a cartoon of it. Its burn size, the distance it reaches, and its final speed are each measured off the running simulation and checked against the answer the problem is solved with by hand: every one lands within 0.08%. The energy books close across both the burn and the collision at the same time.

What else the simulator can model

These three are a sample. The catalog holds 62 published scenes across two families — 19 first-year mechanics and 43 AP-C electromagnetism — every one counted from physics/sim/scenarios/_registry.js and cross-checked against a Node emitter so the number cannot be padded. The embed itself is the vendored app driven by a ?scene= URL parameter behind an accessibility wrapper that respects reduced-motion and pauses when scrolled offscreen.

The receipts — how each number is counted
  • Published scenes — published (published != false) scenes in _registry.js, cross-asserted vs the Node emitter catalog
  • Proved laws — count id="law_NNN" entries in tools/claude_tools/formal_verification/registry.py
  • Property-checked — count classification: fields in docs/formal_verification/codebase_fv_inventory_v1.yaml
  • First-party lines — latest entries[-1].first_party_total.lines in tools/loc_history/history.json

All of these regenerate when tools/build_showcase.py runs, and a deploy-time guard refuses to push a changed figure without an explicit acknowledgement — so a number cannot drift onto the public page unnoticed.
